Getting Started

Requirements

  • Mac with Apple Silicon (M1/M2/M3/M4) for local speech-to-text
  • Python 3.11+ with a virtual environment
  • An Anthropic API key
  • A microphone (built-in or external)

Setup

# Clone and enter the project
cd livemind2

# Create virtual environment and install dependencies
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# Add your API key
echo "ANTHROPIC_API_KEY=sk-ant-..." > .env

Start the server

source .venv/bin/activate
python app.py

The server will ask you to pick a microphone, then start at http://localhost:8765.

First run

  1. Open localhost:8765 in your browser
  2. Optionally enter a meeting topic (helps the AI focus)
  3. Click Connect
  4. Start talking — you'll see live transcription in the sidebar
  5. After ~20 seconds of speech, the first concept map will appear
Tip: The first analysis takes a moment while the AI processes the initial transcript. After that, the map updates every 20 seconds automatically.

Interface Guide

The Map

The main canvas shows a force-directed graph. Each node is a concept extracted from the conversation. Edges show relationships between concepts. Nodes are coloured by category — the AI assigns these automatically.

  • Nodes — represent key concepts, ideas, or topics
  • Edges — show relationships (e.g. "drives", "enables", "part of")
  • Colours — group related concepts by category
  • Labels — concept name (large) and category (small, uppercase)

Status Pill

Top-left corner shows the current state:

LEDMeaning
Green pulseListening — audio is being captured and transcribed
Purple fast pulseMapping — AI is analysing the latest transcript
Amber pulseDegraded — API issue, transcript still flowing, map on hold
GreyDisconnected or paused

Sidebar

  • Transcript — scrolling live text. Latest chunk is highlighted, older text fades. Italic text = partial (still being transcribed).
  • Concepts — chronological list of all concepts that have appeared
  • Controls — Pause, New Session, Reset

Counter

Bottom-left shows the current number of active concepts on the map.

Interactions

Drag nodes

Click and drag any node to reposition it. The physics simulation will settle around the new position. Release to let it float freely again.

Context menu

Right-click (or long-press on trackpad) any node to open the context menu:

ActionWhat it does
Pin / UnpinPinned nodes never get automatically removed, even if the conversation moves on. Their ring becomes thicker.
PromoteBoosts a node's importance score. Useful for reactivating a parked concept or keeping something visible.
RenameChange the label. Opens a prompt dialog.
Merge into...Starts merge mode. Click a second node to merge the first into it. Combines mention counts, redirects all edges. Great for deduplication.
HideImmediately removes the node from the map. It won't come back unless the conversation mentions it heavily again.

Merge mode

After clicking "Merge into...", the source node gets a dashed ring. Click any other node to complete the merge. The source is absorbed into the target — all its connections and history transfer over.

Note: Merge cannot be undone in the current session. The source node is archived permanently.

How the map stays clean

The system automatically manages complexity:

  • Budget: Maximum 24 active nodes at a time. If the conversation generates more, the least important ones are "parked" (removed from view but remembered).
  • Decay: Nodes not mentioned for 12 minutes are automatically parked.
  • Reactivation: If a parked concept gets mentioned 2+ times within 3 minutes, it comes back.
  • Scoring: Every node has an importance score based on how recently it was mentioned (45%), how often (35%), and how connected it is (20%). Pinned nodes get a bonus.

Sessions

What gets saved

Everything. The server stores all session data in a local SQLite database (livemind.db):

  • Every transcript segment (partial and final)
  • Graph snapshots (every 60 seconds + after each AI analysis)
  • All user actions (pins, hides, renames, merges, promotes)
  • Session metadata (topic, start time, end time, summary)

Session controls

ButtonWhat it does
Pause / ResumeStops transcription and AI analysis. The map freezes. Audio capture continues in the background so you won't miss anything when you resume.
New SessionEnds the current session (all data saved), clears the map and transcript, and starts a fresh session. Use this between meetings or topics.
ResetSame as New Session but doesn't create a new one — returns to a blank state. The ended session's data is still in the database.

Reconnecting

If you lose connection (close the tab, browser crash, network hiccup), just reopen the page. If your previous session is less than 4 hours old, you'll see a restore banner on the setup screen. Check the box and click Connect to pick up where you left off.

The server will send back the latest graph snapshot and any transcript segments you missed. The map rebuilds instantly.

Where is the data?

Session data lives in livemind.db in the project directory. This is a standard SQLite file — you can inspect it with any SQLite browser. Tables:

TableContents
sessionsSession ID, topic, timestamps, summary
segmentsEvery transcript chunk with sequence numbers
snapshotsFull graph state at points in time
actionsEvery user interaction (pin, rename, etc.)

Admin Panel

Open localhost:8765/admin to monitor system health in real time.

Key metrics

CardWhat to watch
STT PerformanceSeconds per chunk. The real-time factor should stay below 1.0x (green). Above 1.2x (red) means transcription can't keep up.
Audio BufferSeconds of audio waiting to be processed. Should stay near 0. High values mean lag.
Claude APIResponse time and error count. Errors trigger the circuit breaker.
E2E LatencyTime from audio chunk capture to transcript delivery.
Pipeline StatsChunks processed, silent chunks skipped, partials emitted.
SystemUptime, connected clients, audio levels, VAD state, circuit breaker status.
Graph ChurnNodes added/removed per minute, edge changes, reconnects, frontend FPS.

VAD (Voice Activity Detection)

The server uses energy-based voice activity detection. When no one is speaking, audio chunks are skipped entirely — saving processing time and reducing noise. The admin panel shows whether the VAD thinks someone is currently speaking.

Circuit breaker

If the Claude API returns errors or rate limits, the circuit breaker opens. While open, no API calls are made. It automatically retries after a backoff period (starts at 5s, doubles up to 60s). The admin panel shows the current state: closed (healthy), open (backing off), or half_open (testing recovery).

Troubleshooting

No audio / RMS stays at 0

  • macOS: System Settings → Privacy & Security → Microphone → enable for Terminal
  • System Settings → Sound → Input → increase input volume
  • Make sure you selected the right device when the server started
  • Restart the server with python app.py -d N to force a specific device index

Map not updating

  • Check the status pill — if amber, the API is having issues. It will auto-recover.
  • Make sure your .env file has a valid ANTHROPIC_API_KEY
  • Check the admin panel for Claude API errors
  • The map needs at least 50 characters of new text before triggering an analysis

Browser fallback

If you leave the STT server URL blank on the setup screen (or if the WebSocket connection fails), the system falls back to your browser's built-in speech recognition. This works in Chrome without needing the Python server, but is less accurate and doesn't support partial transcripts.

High latency / audio buffer growing

  • Close other CPU-intensive apps — STT runs on the Neural Engine / GPU
  • Check the admin panel's real-time factor. If consistently above 1.0x, the model can't keep up with real-time audio
  • The server auto-skips ahead if the buffer grows too large (you'll see "catch-up" in the admin stats)

Database

If you need to start completely fresh (wipe all session history), delete livemind.db and restart the server. A new database will be created automatically.
